## §1. 서비스 개요 · 서비스명 후보

**한 줄 정의**: 국립중앙박물관 등 공공 3D 문화유산 데이터를 웹에서 바로 돌려보고, AI 도슨트의 해설을 듣고, 유물의 연고지 여행 정보까지 이어주는 **문화유산 3D 체험 서비스**.

**해결하는 문제**: 공공 3D 문화유산 데이터는 개방되어 있으나 ① 전문 소프트웨어가 필요한 다운로드 파일 형태라 일반인 접근이 어렵고, ② 기관별로 흩어져 있으며, ③ 메타데이터·해설·여행 정보와 분리되어 있다.

**서비스명 후보** (U-05: 6/24까지 확정, 접수 후 변경 불가):

| 후보 | 의미 | 장점 | 유의점 |
|---|---|---|---|
| **손끝유산** | 손끝으로 돌려보는 3D 유물 | 체험성 직관 전달, 검색 고유성 높음 | 영문 표기 어려움 |
| **달빛유람** | 달빛 아래 어디서든 즐기는 유산 야행 (코드명 moon 계승) | 정서·기억성, 야간/원격 향유 서사 | '창덕궁 달빛기행'과 어감 인접 — 혼동 검토 |
| **누리유산** | 공공누리 × 온누리 × 누리집(웹) | 공공데이터 정체성 강조 | 다소 관(官)스러운 어감 |

공통 부제(안): "내 손안의 한국 문화유산 3D"

## §3. 기능 요구사항

우선순위: **P1** 제출 필수(없으면 출품 불가 수준) · **P2** 차별화(심사 가점 직결) · **P3** 여유 시.
AC는 측정·확인 가능해야 하며, 성능 관련 수치는 `01-constitution.md` §3 예산(B1~B4)을 참조한다.

### F1. 유물 3D 뷰어 — P1

> P-A/B/C: "유물을 모든 각도에서 자세히 살펴보고 싶다."

- **AC-F1-1** 마우스 드래그·터치 스와이프로 360° 회전, 휠·핀치로 줌, 원위치 리셋 버튼이 동작한다.
- **AC-F1-2** 모델 로딩 중 진행률(%)이 표시된다. (B2b 연동)
- **AC-F1-3** WebGL2 미지원 환경에서는 폴백 정적 이미지와 안내 문구가 표시되고, 메타데이터는 전부 접근 가능하다. (§7 헌법 연동)
- **AC-F1-4** 풀스크린 토글이 동작한다 (수업 빔프로젝터 시나리오).
- **AC-F1-5** 자동회전 켜기/끄기 옵션이 있다 (기본: 꺼짐 — 멀미 배려).
- **AC-F1-6** 성능 예산 B1·B2a·B2b를 충족한다.
- **AC-F1-7** "실제 크기" 토글 시 유물을 실측 치수로 표시하고 **사람 실루엣(≈170cm)+눈금자**로 크기를 대조한다(치수 정보가 있는 유물). (`src/experience/scale.ts`)

### F2. 유물 카탈로그 — P1

> P-A: "수업 주제(시대·재질)에 맞는 유물을 빨리 찾고 싶다."

- **AC-F2-1** 유물 **6점 이상**이 그리드(썸네일·명칭·시대)로 표시된다.
- **AC-F2-2** 명칭 검색 입력 시 즉시(추가 페이지 이동 없이) 결과가 좁혀진다.
- **AC-F2-3** 시대 · **분류(유물 특징)** · 재질 필터를 조합할 수 있다. 시대 옵션은 연대순으로 정렬된다.
- **AC-F2-3b** 결과를 **시대순 / 분류별**로 묶어 섹션 헤딩(분류·점수)과 함께 볼 수 있다. (분류 체계: `src/catalog/taxonomy.ts`)
- **AC-F2-5** 유물은 **3D 모델 / 이미지 자료** 두 유형이 공존하며 카드 배지로 구분되고 유형 필터로 좁힐 수 있다. (이미지 유물 = 초기 시대 보강분, `media:"image"`)
- **AC-F2-4** 결과 없음 상태가 명시적으로 표시된다.

### F3. 유물 상세 · 출처표시 — P1

> 모든 페르소나: "이 유물이 무엇인지, 어디 소장품인지 알고 싶다."

- **AC-F3-1** 명칭·시대·재질·크기·소장처·설명이 표시된다. 누락 필드는 "정보 없음"으로 명시한다.
- **AC-F3-2** **모든 유물(100%)**에 공공누리(KOGL) 유형 배지 + 제공기관 + 원천 URL 링크가 표시된다. (헌법 §1)
- **AC-F3-3** e뮤지엄 API 유래 메타데이터는 데이터 출처 필드로 식별 가능하다 (데이터활용 증빙).

### F4. AI 도슨트 — P1 ★2026 격상

> 격상 근거: 1차 심사 **AI활용 20%** + 부문 정의 "AI 기술도 포함한 제품/서비스" (`00` §2·§6)
> P-A/C: "이 유물에 대해 궁금한 것을 물어보고 쉬운 해설을 듣고 싶다."

- **AC-F4-1** 유물 상세에서 해당 유물 맥락의 대화형 해설을 제공하며, 응답은 스트리밍으로 표시된다 (첫 토큰 ≤ 3초).
- **AC-F4-2** 유물별 추천 질문 칩이 3개 이상 제공된다 (예: "어디서 발견됐나요?", "무엇에 쓰던 물건인가요?").
- **AC-F4-3** 답변은 주입된 유물 메타데이터에 근거하며, 근거 없는 단정을 피하도록 가드레일 프롬프트를 적용한다. 응답 영역에 "AI 생성" 고지를 표시한다. (헌법 §1-4)
- **AC-F4-4** 남용 방지 장치가 있다: IP 레이트리밋 + 세션당 메시지 상한 + 응답 토큰 상한. 키 미설정 시 mock 제공자로 UI가 정상 동작한다.
- **AC-F4-5** **제출 시점에는 실 LLM이 연결**되어 있어야 한다 (mock 상태로 제출 금지 — U-04 의존).

### F5. 관광지 연계 — P2

> P-B: "이 유물과 연고가 있는 곳에 직접 가보고 싶다." — 데이터활용 25%(TourAPI 융합) 대응

- **AC-F5-1** 연고지가 있는 유물 상세에 지도와 연고지 마커가 표시된다.
- **AC-F5-2** TourAPI 기반 주변 관광지가 3건 이상(명칭·거리·대표 이미지) 표시된다.
- **AC-F5-3** API 실패 시 사전 수집 스냅숏으로 폴백하고, 스냅숏 수집 시각을 표기한다.
- **AC-F5-4** "한국관광공사 TourAPI" 출처가 표기된다.

### F6. 가상 전시관 — P2

> P-C: "박물관을 거니는 기분으로 둘러보고 싶다." — 혁신성·독창성 대응

- **AC-F6-1** 3D 전시실이 **분류(유물 특징)별 구역(zone)**으로 나뉘고, 각 구역에 분류 표지가 붙으며 구역 안은 시대순으로 배치된다(총 4점 이상).
- **AC-F6-2** **키보드 1인칭 보행**(W·A·S·D 또는 화살표로 이동, ←→·Q·E로 회전)으로 전시실을 거닐 수 있고, 유물에 다가가 **Enter** 또는 클릭으로 해당 상세 페이지로 이동한다. 터치 환경용 온스크린 조작 버튼을 제공한다.
- **AC-F6-3** 입장 전 총 용량 안내 + 조작 안내가 표시되고, 입장 시 로딩 진행률이 표시된다.
- **AC-F6-4** 구역 배치는 등록·발행 유물에서 자동 생성되어(저장소 + 분류 체계 파생) 유물 추가 시 자동 반영된다. 한 씬 과부하 방지를 위해 분류별 대표 N점만 로드한다.
- **AC-F6-5** 유물별 스포트라이트·천장·발광 구역 띠·안개로 박물관 분위기를 연출한다(환경맵 CDN 비의존, 오프라인 안정).

### F7. 공유·임베드 — P3

> P-A: "수업 자료에 링크/임베드로 넣고 싶다."

- **AC-F7-1** 유물별 OG 메타태그(제목·설명·대표 이미지)가 생성된다.
- **AC-F7-2** `/embed/[slug]` 라우트가 뷰어 단독 화면을 제공한다.

### F9. 시대 타임라인 — P2 ★신설

> "시대별로 유물을 보며 그 시대의 배경을 알고 싶다." — 활용성(교육)·독창성 대응

- **AC-F9-1** 구석기~조선을 연대순 타임라인으로 제시하고, 시대를 펼치면 **시대 개요·생활상(편집 요약)**과 해당 시대 유물(3D·이미지 혼합)이 함께 표시된다.
- **AC-F9-2** 유물 카드를 클릭하면 상세로 이동한다. 시대 점프 칩으로 빠르게 이동한다.
- **AC-F9-3** 시대 개요는 "편집 요약"임을, 유물·이미지 출처는 국립중앙박물관임을 명시한다(정직성).
- **AC-F9-4** 각 시대 **"생활 장면"**: 양식화 배경(SVG) 위에 **실제 유물 이미지**를 배치하고, 클릭 시 그 유물의 **쓰임새(어떻게 썼는지)**를 보여준다. 배경이 양식화 그림임을 명시한다. (`src/timeline/EraScene.tsx`·`content/era-scenes.json`)
- **AC-F9-5** 대표 유물의 쓰임새(`usage`)는 상세 페이지 "쓰임새" 섹션과 AI 도슨트 컨텍스트에도 반영된다(F4 연계).

### F8. AI 큐레이션 노트 — P3

> 빌드타임에 LLM으로 생성하고 **사람이 검수**한 짧은 감상 포인트. F4의 보완(AI활용 증빙 다변화).

- **AC-F8-1** 노트가 있는 유물 상세에 감상 포인트 2~3개가 표시되고 "AI 생성·검수됨" 고지가 붙는다.
- **AC-F8-2** (선택) AI 생성 역사 장면 이미지(디퓨전, Z-Image)를 스토리 표지 등에 쓰되 **"AI 생성 재현"** 라벨을 붙인다. ⚠️ 생성은 HF 토큰 필요(미설정 시 실유물 포스터로 폴백).

### F10. 역사 이야기(스크롤리텔링) — P2 ★신설

> "개별 유물이 아니라 흐름이 있는 역사를 흥미롭게." — 혁신성·독창성·활용성

- **AC-F10-1** 스크롤에 따라 **sticky 무대(3D/이미지)가 전환**되며 한국사 한 주제를 들려준다(예: "흙에서 불로 — 그릇 1만 년"). 진행 표시·유물 상세 링크.
- **AC-F10-2** 본문은 편집 글임을, 유물 출처는 박물관임을 명시한다.

### F11. AI 의미 검색 + 비슷한 유물 — P1 ★AI활용 다변화

> 프롬프트 LLM 외 **임베딩(벡터) AI**. `transformers.js`(multilingual-e5-small) **온디바이스·키 불필요**.

- **AC-F11-1** 상세에 **"비슷한 유물"**(사전계산 임베딩 이웃)이 표시된다.
- **AC-F11-2** `/search`에서 **뜻으로** 유물을 찾는다. 주제 칩은 사전 임베딩으로 즉시, 자유 입력은 브라우저 모델 지연 로드(검색어 서버 미전송). 실패 시 키워드 폴백.

### F12. 유물 퀴즈·게임 — P2

> 즐기며 배우는 게임화. 메타데이터 + **임베딩으로 오답·정답 선별**.

- **AC-F12-1** 시대 맞히기·이름 맞히기·**가장 비슷한 유물 고르기**(임베딩) 문항을 라운드제로 풀고 점수·해설·유물 링크를 제공한다.

## §5. 범위 제외 (Out of Scope)

로그인·회원, 댓글 등 UGC, 네이티브 앱, 관리자 CMS, 결제. — 이유: 14일 일정 + 우수사례 평가에 불필요.
**발전가능성 로드맵으로만 언급**(기획서용): 다국어(영어), WebXR(VR 관람), 유물 수 확대(저장소 패턴 → DB 전환), 학교 수업 패키지.
