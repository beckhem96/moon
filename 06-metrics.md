## 3. B2 — 로딩 시간 (진입 → 3D 인터랙션 가능)

> 측정: 헤드리스 Chrome(`scripts/measure-load.mjs`). 캐시 비활성, 페이지 진입부터 모델 렌더 완료 신호(`data-model-ready`)까지. 3회 중앙값. Fast 4G = 4Mbps↓/3Mbps↑/20ms(Chrome DevTools 근사).

| 유물 | 광대역(B2a) | Fast 4G(B2b) |
|---|---|---|
| 금동 반가사유상 (1.27MB) | 0.54초 | 6.00초 |
| 평양 석암리 띠고리 (1.6MB·최대) | 0.52초 | 6.73초 |

- **B2a** 목표 ≤3초: 광대역 0.5초대로 큰 여유.
- **B2b** 목표 ≤8초: 최대 용량 유물조차 6.7초로 충족. 로딩 진행률(%) 표시로 체감 보완(AC-F1-2).
- 평균 용량(0.99MB) 유물은 Fast 4G에서 ~5초 수준으로 추정.

## 4. 측정 재현 방법

```bash
cd web
npm run build && npm run start        # 로컬 프로덕션 서버(:3000)
node scripts/measure-load.mjs         # B2 로딩 시간(광대역·Fast 4G, 3회 중앙값)
node -e "console.log(require('./content/metrics.json'))"  # B1·B4 원천
```

- B1·B4는 에셋 파이프라인이 발행 시 자동 기록 → 수작업 개입 없음(헌법 §3).
- B3(Lighthouse)는 배포 후 `홈·카탈로그·상세` 각 3회 중앙값으로 측정해 본 문서에 추가.
