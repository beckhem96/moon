# 01. 프로젝트 헌법 (Constitution)

> 작성일: 2026-06-12 · 상태: **채택** · 선행: `00-contest-brief.md` · 후행: `02-spec.md` `03-domain.md` `04-plan.md` `05-tasks.md`
> 프로젝트 전 기간(~2026-06-26) 동안 유지되는 불변 원칙. 모든 하위 문서·코드는 본 헌법을 위반할 수 없다. 개정은 §8 절차로만 가능.

## §0. 미션

> **"공공 3D 문화유산 데이터를 '다운로드용 파일'에서 '누구나 웹에서 즐기는 체험 콘텐츠'로."**

흩어져 있는 공공 3D 원천데이터를 통합 큐레이션하고, 유물 메타데이터·관광정보·AI 해설과 융합해 시공간 제약 없는 한국 문화유산 탐방 경험을 제공한다. (코드명: **moon** — 서비스명은 `02-spec.md` §1 후보에서 6/24까지 확정)

## §1. 데이터 출처표시 의무 — 탈락 방지 조항

1. 모든 유물 레코드는 **공공누리(KOGL) 유형 · 제공기관 · 원천 URL**을 필수 필드로 보유한다. 셋 중 하나라도 없으면 등록 불가.
2. 출처는 화면에 노출한다: 유물 상세(개별), 가상 전시관(개별), `/about`(전체 데이터 출처 일람).
3. 라이선스가 불명확한 데이터(예: 국가유산청 일부 항목)는 **확인 전 사용 금지**.
4. AI 도슨트 응답·생성 콘텐츠에도 "AI 생성" 고지를 표기한다.

## §2. 실배포 우선 — 완성도 30% 대응

1. 우수사례 부문은 "완성된 제품·서비스"를 평가한다. 데모가 아니라 **실서비스**를 만든다.
2. `main`은 항상 Vercel에 배포 가능한 상태를 유지한다. 깨진 상태로 하루를 넘기지 않는다.
3. **실데이터만 사용한다.** 가짜 유물·로렘 입숨 금지. 외부 API 폴백도 실데이터 스냅숏(수집 시각·출처 명기) 우선.

## §3. 정량 지표 의무 — 성능 예산

아래 예산을 위반하는 에셋·페이지는 등록/머지할 수 없다. 측정 방법은 `04-plan.md` §11.

| ID | 항목 | 목표 | 상한 |
|---|---|---|---|
| B1 | 유물당 발행 GLB 용량 | ≤ 5MB | 8MB |
| B2a | 유물 상세 진입 → 3D 인터랙션 가능 (광대역 ≥30Mbps) | ≤ 3초 | 5초 |
| B2b | 〃 (Chrome Fast 4G 스로틀) | ≤ 8초 | 12초 · 로딩 진행률 표시 필수 |
| B3 | Lighthouse 모바일 Performance | 홈·카탈로그 ≥ 85 · 유물 상세 ≥ 70 | — |
| B4 | 원본 대비 용량 절감률 (압축 파이프라인) | ≥ 90% | — |

- 에셋 파이프라인은 단계별 지표(용량·폴리곤 수·압축률)를 **자동 기록**한다 → `06-metrics.md` 증빙의 원천.
- 지표는 심사 대응 수단이기 이전에 사용자 경험의 하한선이다.

## §4. 범위 규율

1. 우선순위 체계: **P1(제출 필수) > P2(차별화) > P3(여유 시)**. 정의는 `02-spec.md` §3.
2. P1이 미완인 동안 P2에 착수하지 않는다 (단일 작업 스트림 기준).
3. **2026-06-22(월) 00:00부터 기능 동결** — 신규 기능 금지. 버그픽스·콘텐츠 등록·문서·지표 측정만 허용.
4. 일정 위험 시 컷 결정은 `05-tasks.md` §4 컷 라인 규칙을 따른다. 즉흥 컷 금지.

## §5. SDD 규칙 — 문서가 진실의 원천

1. 문서(00~05)와 구현이 어긋나면 **문서를 먼저 갱신**하고 코드를 바꾼다.
2. 기능 추가·변경은 `02-spec.md`에 수용 기준(AC)을 먼저 적은 뒤 구현한다.
3. ⚠️ 마커는 "미확인 외부 사실"을 뜻한다. 확인 즉시 마커를 제거하고 갱신 날짜를 남긴다.
4. 용어는 `03-domain.md` 용어집을 따른다. 코드 식별자·문서·UI 문구가 같은 개념에 다른 이름을 쓰지 않는다.

## §6. 스택 고정

1. **Next.js(App Router) + TypeScript(strict) + React Three Fiber + Tailwind CSS**, 배포는 **Vercel**. (사용자 확정 2026-06-12)
2. **DB를 도입하지 않는다** — 정적 콘텐츠 모듈 + 저장소 패턴 추상화. 근거와 확장 경로는 `04-plan.md` §3.
3. **LLM 제공자는 미정** — `DocentProvider` 포트 뒤로 격리하고 mock을 기본 제공, 구현체 교체는 1파일 추가로 가능해야 한다. (U-04, 6/17까지 결정)
4. 신규 런타임 의존성 추가 시 `04-plan.md` §2 스택 표에 근거 한 줄을 추가해야 한다.

## §7. 사용자 경험 원칙 — 사회적기여·ESG 대응이자 서비스 본질

1. **모바일 우선** (뷰포트 360px~), **한국어 우선**.
2. 접근성: 뷰어 키보드 조작, 이미지 대체 텍스트, WCAG AA 명도 대비, 자동회전 정지 옵션(멀미 배려).
3. 3D 미지원 환경에서도 콘텐츠에 접근 가능해야 한다 — 폴백 이미지 + 전체 메타데이터.
4. 위 항목은 심사 대응이기 이전에 미션(§0) "시공간 제약 없는 향유"의 구현이다.

## §8. 거버넌스

1. 헌법 개정은 (a) 하위 문서와의 충돌 발견, (b) 일정상 실현 불가 판명 시에만, 사유와 날짜를 본 문서에 기록하고 수행한다.
2. 매일 작업 시작 시 `05-tasks.md` §5 일일 체크인을 갱신한다.
3. 제출물(신청서·기획서)과 본 문서 체계가 어긋나면 제출물 기준으로 문서를 정정한다 (접수 후 신청 내용 변경 불가이므로).
